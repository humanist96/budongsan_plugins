# 튜터리얼 6: 지역비교 분석 리포트 자동생성

## 개요

- **목적:** 여러 지역의 부동산 데이터를 수집하고 비교 분석하여, 투자 의사결정을 위한 종합 리포트를 자동으로 생성합니다.
- **대상 독자:** 투자 지역 선정에 고민하는 부동산 투자자
- **소요 시간:** 약 60-70분
- **난이도:** 중급-고급
- **사용 플러그인:** mcp-kr-realestate, firecrawl-mcp-server, mcp-perplexity, data:analyze

이 튜터리얼에서는 서울 3개 구(마포구, 성동구, 동작구)를 대상으로 실거래가, 인프라, 개발호재 등 다차원 데이터를 수집한 뒤, 정량적 비교 매트릭스와 투자 추천 리포트를 자동 생성하는 전체 워크플로우를 다룹니다.

---

## 사전 준비

### 필수 설정

1. **mcp-kr-realestate 설정 완료** (튜터리얼 1 참조)
   - 국토교통부 공공데이터 API 키 발급 및 등록
   - 아파트 매매/전세 실거래가 조회 가능 상태 확인

2. **Firecrawl 설정 완료** (튜터리얼 2 참조)
   - Firecrawl API 키 발급 및 MCP 서버 등록
   - 웹 크롤링 테스트 완료

### 선택 설정 (권장)

3. **Perplexity API 키 발급**

Perplexity MCP는 AI 기반 웹 검색을 통해 최신 정책 변경, 재개발 뉴스 등을 빠르게 수집할 수 있습니다.

`.claude/settings.local.json` 또는 프로젝트의 `.mcp.json`에 다음을 추가합니다:

```json
{
  "mcpServers": {
    "perplexity": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-perplexity"],
      "env": {
        "PERPLEXITY_API_KEY": "${PERPLEXITY_API_KEY}"
      }
    }
  }
}
```

환경변수를 설정합니다:

```bash
export PERPLEXITY_API_KEY="pplx-xxxxxxxxxxxxxxxxxxxxxxxx"
```

> **참고:** Perplexity API가 없더라도 firecrawl-mcp-server를 활용한 웹 검색으로 대체할 수 있습니다. 다만, Perplexity를 사용하면 여러 출처의 정보를 AI가 요약해주므로 리서치 시간이 크게 단축됩니다.

### 비교 대상 지역 사전 조사

본 튜터리얼에서는 다음 3개 구를 비교합니다:

| 구분 | 마포구 | 성동구 | 동작구 |
|------|--------|--------|--------|
| 위치 | 서울 서북부 | 서울 동부(한강변) | 서울 남부(한강변) |
| 특징 | 홍대/합정 상권, 여의도 접근성 | 성수동 핫플레이스, 강남 접근성 | 흑석/사당 학군, 9호선 |
| 84m2 평균 시세 | 약 13억 원 | 약 14억 원 | 약 11억 원 |

## Step 1: 비교 대상 지역 선정

### Claude Code에 분석 요청

터미널에서 Claude Code를 실행하고 다음 프롬프트를 입력합니다:

```text
마포구, 성동구, 동작구를 부동산 투자 관점에서 비교 분석해줘.
비교 기준은 다음과 같아:
1. 현재 시세 및 가격 추이
2. 전세가율
3. 거래량 추이
4. 교통 접근성
5. 학군
6. 생활 인프라
7. 개발 호재
8. 가격 매력도 (강남 대비 할인율)

먼저 각 지역의 기본 개요를 정리하고, 이후 단계별로 상세 데이터를 수집하자.
```

### 예상 응답

Claude Code는 먼저 각 지역의 기본 프로필을 정리합니다:

```text
## 지역 기본 프로필

### 마포구
- 면적: 23.84km², 인구: 약 37만 명
- 주요 생활권: 합정/망원, 홍대/연남, 공덕/마포, 상암
- 대표 아파트: 마포래미안푸르지오, 마포프레스티지자이, 신촌숲아이파크
- 교통: 2호선, 5호선, 6호선, 경의중앙선, 공항철도
- 핵심 키워드: 여의도 접근성, 상암 디지털미디어시티, 홍대 상권

### 성동구
- 면적: 16.86km², 인구: 약 30만 명
- 주요 생활권: 성수, 왕십리, 금호/옥수, 행당/응봉
- 대표 아파트: 서울숲트리마제, 옥수하이츠, 왕십리텐즈힐
- 교통: 2호선, 3호선, 5호선, 분당선, 경의중앙선
- 핵심 키워드: 성수동 핫플레이스, 강남 접근성, 한강 조망

### 동작구
- 면적: 16.35km², 인구: 약 38만 명
- 주요 생활권: 흑석, 사당, 노량진, 상도
- 대표 아파트: 흑석한강센트레빌, 이수브라운스톤, 상도파크자이
- 교통: 4호선, 7호선, 9호선
- 핵심 키워드: 흑석 뉴타운, 9호선 급행, 서울대 학군
```

> **팁:** 이 단계에서 Claude Code가 mcp-kr-realestate를 자동 호출하여 기본 시세를 함께 보여줄 수 있습니다. 자동 호출이 되지 않으면 Step 2에서 명시적으로 요청합니다.

## Step 2: 실거래가 데이터 수집

### 3개 구 동시 조회

다음 프롬프트로 실거래가 데이터를 수집합니다:

```text
mcp-kr-realestate를 사용해서 마포구, 성동구, 동작구의 아파트 실거래가 데이터를 조회해줘.

조회 조건:
- 기간: 2023년 1월 ~ 2025년 12월 (최근 3년)
- 대상: 전용면적 84m2 기준 아파트
- 필요 데이터: 평균 매매가, 평균 전세가, 월별 거래량

3개 구의 데이터를 테이블로 비교해줘.
```

Claude Code는 mcp-kr-realestate 플러그인을 통해 국토교통부 실거래가 데이터를 조회합니다. 내부적으로 각 지역코드(마포구: 11440, 성동구: 11200, 동작구: 11590)를 사용하여 API를 호출합니다.

## Step 3: 인프라/교통/학군 정보 크롤링

### Firecrawl을 활용한 정보 수집

다음 프롬프트로 각 지역의 인프라 정보를 수집합니다:

```text
firecrawl을 사용해서 마포구, 성동구, 동작구의 생활 인프라 정보를 수집해줘.

수집 항목:
1. 교통: 지하철역 수, 노선 수, 주요 버스 노선, 여의도/강남/도심 접근 시간
2. 학군: 초/중/고등학교 수, 학업 성취도 상위 학교
3. 생활 인프라: 대형마트, 종합병원, 대형 공원, 주요 상권
4. 문화시설: 도서관, 체육시설, 문화센터

결과를 비교 테이블로 정리해줘.
```

Claude Code는 firecrawl-mcp-server를 통해 각 구청 홈페이지, 부동산 정보 사이트, 교육청 자료 등에서 관련 정보를 크롤링합니다.

## Step 4: 개발 호재 및 정책 리서치

### Perplexity MCP 활용

Perplexity API가 설정되어 있다면, AI 기반 웹 검색으로 최신 개발 호재와 정책 변화를 조사합니다:

```text
perplexity를 사용해서 마포구, 성동구, 동작구의 부동산 관련 개발 호재를 조사해줘.

조사 항목:
1. 재개발/재건축 계획 및 진행 현황
2. GTX, 신규 지하철 노선 등 교통 개발 계획
3. 도시재생 뉴딜 사업
4. 대규모 개발 사업 (복합개발, 상업지구 조성 등)
5. 최근 1년 내 주요 정책 변화

각 호재의 예상 완료 시점과 부동산 가격 영향도를 함께 분석해줘.
```

### Perplexity API가 없는 경우 (대안)

firecrawl을 활용하여 부동산 뉴스 사이트를 직접 크롤링합니다:

```text
firecrawl을 사용해서 다음 사이트들에서 마포구, 성동구, 동작구 개발 호재 뉴스를 수집해줘:
- 네이버 부동산 뉴스
- 매일경제 부동산 섹션
- 각 구청 도시개발 공지사항

최근 6개월 이내 기사를 중심으로 정리해줘.
```

### 예상 개발 호재 분석 결과

```text

### 마포구
```

```text
서울 주요 재건축 단지를 진행 단계별로 비교해줘.

비교 대상:
1. 초기 단계: 안전진단 통과 또는 준비 중인 단지
2. 중기 단계: 조합 설립 ~ 사업시행인가 단계
3. 후기 단계: 관리처분 ~ 착공 단계

각 단계별 투자 수익률, 리스크, 예상 투자 기간을 비교하고,
현재 시점에서 가장 매력적인 재건축 투자 단지 Top 5를 추천해줘.
```

재건축 투자 비교 시 핵심 점검 사항:
- 용적률 상향 가능성 (기존 용적률 대비 허용 용적률)
- 조합원 분담금 추정액
- 일반 분양 수익성
- 사업 지연 리스크 (주민 합의율, 인허가 이슈)

### 응용 3: 월세 수익형 투자 지역 비교

임대 수익률 기준으로 지역을 비교합니다:

```text
서울에서 월세 수익형 투자에 적합한 지역을 비교해줘.

비교 대상: 관악구, 마포구, 성북구, 동대문구, 영등포구

분석 기준:
1. 소형 평형(59m2 이하) 매매가 대비 월세 수익률
2. 임차 수요 (대학가, 역세권, 오피스 밀집 지역)
3. 공실률
4. 임대료 상승 추이
5. 관리 편의성 (신축 비율, 관리 인프라)

연 수익률 5% 이상 달성 가능한 지역과 적정 매입가를 제시해줘.
```

수익형 투자 비교 시 추가 고려 사항:
- 세금 (종합소득세, 건강보험료 증가)
- 대출 이자 대비 순수익률
- 임차인 유형별 리스크 (학생, 직장인, 외국인)

## 리포트 품질 향상 팁

### 1. 데이터 출처를 명시하여 신뢰도 확보

리포트 작성 시 다음 프롬프트를 추가하면 출처가 명시됩니다:

```text
분석 리포트에 모든 데이터의 출처를 각주로 표시해줘.
형식: [출처] 국토교통부 실거래가 공개시스템, 2025년 12월 기준
```

주요 공신력 있는 출처:
- **실거래가:** 국토교통부 실거래가 공개시스템 (rt.molit.go.kr)
- **공시가격:** 국토교통부 부동산공시가격 알리미
- **인구통계:** 행정안전부 주민등록 인구통계
- **개발계획:** 서울시 도시계획포털, 각 구청 도시개발과
- **학군정보:** 학교알리미 (schoolinfo.go.kr)

### 2. 정성적 분석 + 정량적 분석 균형

수치 데이터만으로는 투자 판단이 어렵습니다. 다음과 같이 정성적 분석을 보강합니다:

```text
각 지역의 현장 분위기, 주민 만족도, 향후 발전 잠재력에 대한
정성적 평가도 추가해줘. 특히:
- 거리 분위기 (활기, 안전성, 깨끗함)
- 주민 연령대 및 커뮤니티 특성
- 주변 지역과의 시너지 효과
```

### 3. 리스크 요인을 반드시 포함

투자 추천만 있고 리스크 분석이 없는 리포트는 불완전합니다:

```text
각 지역의 투자 리스크를 반드시 포함해줘.
- 최악의 시나리오 (가격 하락 시 예상 손실)
- 유동성 리스크 (급매 시 할인율)
- 정책 리스크 (규제 강화 시 영향)
- 시장 리스크 (금리 변동, 경기 침체)
```

### 4. 날짜/시점 명시

부동산 데이터는 시점에 따라 크게 달라지므로, 모든 수치에 기준 시점을 명시합니다:

```text
모든 가격 데이터에 기준 시점을 명시해줘.
예: "84m2 평균 매매가 13억 원 (2025년 12월 기준)"
```

### 5. 시나리오별 분석 추가

단일 전망이 아닌 복수 시나리오를 제시하면 리포트의 활용도가 높아집니다:

```text
향후 3년 시세 전망을 낙관/기본/비관 3가지 시나리오로 제시해줘.

- 낙관: 금리 인하 + 규제 완화 + 개발 호재 조기 실현
- 기본: 현재 추세 유지
- 비관: 금리 인상 + 규제 강화 + 개발 호재 지연
```

## 자주 발생하는 문제 해결

### 문제 1: 실거래가 데이터 조회 실패

```text
Error: API 호출 실패 - 일일 호출 한도 초과
```

**해결 방법:**
- 국토교통부 API는 일일 1,000건 호출 제한이 있습니다.
- 3개 구를 한 번에 조회하면 호출 횟수가 빠르게 소진됩니다.
- 조회 기간을 분할하거나 (1년씩), 다음 날 이어서 분석하세요.

### 문제 2: Firecrawl 크롤링 속도 저하

```text
일부 페이지 크롤링 타임아웃
```

**해결 방법:**
- 크롤링 대상 URL을 구체적으로 지정합니다.
- 한 번에 너무 많은 페이지를 크롤링하지 않습니다.
- 필요한 정보만 선별적으로 수집합니다.

### 문제 3: Perplexity API 응답이 부정확한 경우

**해결 방법:**
- 질문을 구체적으로 작성합니다: "마포구 재개발 현황" 대신 "마포구 공덕동 재정비촉진구역 2025년 진행 현황"
- 검색 결과를 firecrawl로 교차 검증합니다.
- 공식 출처(구청 홈페이지, 정비사업 조합 공지)를 우선 참조합니다.
